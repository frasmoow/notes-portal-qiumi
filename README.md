# notes-portal-qiumi

本仓库用于归档和发布多个独立 HTML 页面，作为笔记与静态内容的统一入口。

## 项目简介

这是一个以 HTML 页面为主要载体的归档仓库，主要用于存放各类独立页面。每个页面均为自包含的 HTML 文件，不依赖特定后端服务或数据库。仓库本身不针对任何特定网站或域名，仅作为页面资料的集合与发布渠道。

## 目录说明

```
.
├── index.html          # 入口页面（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example1.html
│   └── example2.html
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面，每个页面均可直接访问。
- `assets/`：存放页面引用的公共静态资源，按类型分目录存放。

## 页面归档说明

- 所有页面均为纯前端 HTML，可直接在浏览器中打开或通过 GitHub Pages 等服务部署。
- 页面内容不涉及任何具体域名、网站或商业服务，仅作为技术笔记或静态内容展示。
- 页面命名建议采用清晰、无歧义的英文或拼音，并尽量保持目录结构简洁。

## 维护说明

- 添加新页面时，请将 HTML 文件放入 `pages/` 目录，并确保引用的资源路径正确。
- 如需更新已有页面，请直接替换或修改对应文件，并保持版本控制记录清晰。
- 不建议在页面中嵌入外部敏感链接或动态内容，以保持仓库的通用性与可迁移性。
- 欢迎通过 Pull Request 或 Issue 提交改进建议或新增页面。
